01 — SEO Experiments

Building to Learn

I built four live websites to test how search engines index and rank content — each one designed around a specific hypothesis.

01 Local SEO

Textile Website

A hyper-local SEO experiment targeting search queries like "tailoring in Guduvanchery" to test how geo-specific content ranks.

sree-amman-textiles.netlify.app
Visit Website

Goal

Rank a local tailoring business for nearby search terms and observe how geographic relevance affects visibility.

Tools Used

Google Search Console · SEO fundamentals · HTML meta tags · XML sitemap

What I Did

Targeted geo-specific keywords, optimised page titles and meta descriptions, submitted a sitemap, and monitored indexing status via Search Console.

What I Observed

The site began appearing in Google results for local tailoring queries within weeks of submission. Impressions grew steadily once indexing completed.

Key Learning

Local SEO works — but only when keywords match the exact phrases real users type. Specificity beats broad terms.

02 EMD Experiment

CloudAdmin Website

An exact match domain (EMD) experiment to test whether domain names themselves carry a ranking advantage over content quality.

cloudadmin.live
Visit Website

Goal

Understand if choosing a domain that exactly matches a target keyword gives a measurable search ranking signal.

Experiment

Registered an exact-match domain and published minimal content. Tracked indexing time, SERP position, and impression growth over several weeks.

What I Observed

The domain indexed quickly but did not automatically outrank established competitors. Content quality and backlink signals still dominated ranking.

Key Learning

EMDs can give a minor relevance hint, but domain name alone cannot compensate for weak content. Substance always beats shortcuts.

03 No-code SEO

WebEase Website

Testing how well Zoho Sites — a no-code builder — supports on-page SEO without writing a single line of backend code.

webease.zohosites.in
Visit Website

Goal

Evaluate Zoho Sites' built-in SEO tools — meta tag controls, URL structure, page speed, and search console integration.

Tools

Zoho Sites builder · Google Search Console · On-page SEO settings within the platform

Key Learning

No-code builders offer convenience but impose limitations. Zoho Sites handles basic SEO well, but advanced optimisations require custom code access. Platform choice matters early.

04 Event Marketing

Science Station Website

Built for the Zoho School trade show — combining web design with real event and product marketing in a miniature launch scenario.

mmegakalai19-bit.github.io/science-station
Visit Website

Goal

Create a website that supports a real trade show event — communicating the product, generating interest, and acting as a digital touchpoint for visitors.

Marketing Context

The site functioned as a product landing page during the event. Visitors could explore offerings, understand value props, and connect digitally — simulating real product launch behaviour.

Key Learning

A website is a marketing asset, not just a technical output. Clear messaging, fast loading, and a compelling CTA matter just as much as design quality.

03 — Insights

What I Learned

Six observations from running live SEO experiments — not theory.

01

Intent beats keywords

Match the reason someone searches, not just the word.

02

Domain name ≠ ranking

EMDs give a minor hint. Content quality still decides.

03

Relevance is non-negotiable

Thin content doesn't rank, regardless of other signals.

04

Indexing takes patience

Impressions don't appear overnight — monitor via GSC.

05

Local SEO = geo keywords

City names in titles and headers drive local visibility.

06

Platform limits your ceiling

Pick your builder knowing its SEO constraints upfront.
